Main Components

The framework's main interfaces are the components GlobalDataLayer and UILayer. GlobalDataLayer contains all React contexts of the framework to share data across the whole component tree. The UILayer component contains the components which actually render the UI.

The reason for the separation into two layers can be understood by the following example: Imagine developing a React context which needs the authentication information. The authentication information is stored inside the GlobalDataLayer. Where to put your React context's provider? To pass your context to the UILayer and then rendering the Provider seems inappropriate. Instead, you put the Provider between the GlobalDataLayer and UILayer yourself. By doing this, your React context has access to all of the framework's contexts, and the UILayer additionally has access to your context.

An example for this situation:

...
return (
// If you don't want to use the default authentication provider, you would have to render another authentication provider at this position. Authentication providers will be explained later on.
    <GlobalDataLayer ...properties...>
    // Your react contexts go here.
        <UILayer ...properties... />
    </GlobalDataLayer>
);
...

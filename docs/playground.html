Cloning the Repository

If you want to play with the framework and the contained example project, you have to clone the framework's Git repository first.

The Example Project

As mentioned, the repository contains the framework and an example project. As the framework isn't a UI by itself, you cannot see the consequences of framework changes without a project using the framework. This is where the example project comes into use. The example project uses the locally built framework as a file dependency in order to visualize the consequences of framework changes.

Setting Up and Running the Example Project

To start the example project you have to execute npm run setup_and_build inside the root folder of the repository. This will setup the environment and start the development server at port 3000.

Making Real-Time Changes to the Framework

If you want to make changes to the framework and see the consequences of these changes in real time you have to execute the following steps by yourself:

1. Execute npm install inside the repository's root folder.
2. Execute npm run devmode inside the repository's root folder. The script will watch for file changes of the framework's source code and recompile if needed.
3. Open another terminal inside the example project's (example folder) folder.
4. Execute npm install inside the example project's (example folder) folder.
5. Execute npm run dev inside the example project's (example folder) folder.
